Scope of the document

This Algorithm Theoretical Basis Document (ATBD) describes the atmospheric correction (AC) processing chain for deriving Top-of-Canopy (TOC) directional reflectances from Top-of-Atmosphere (TOA) reflectances measured by the Proba-V/VGT sensors. The processing implements rigorous uncertainty propagation following the FIDUCEO (Fidelity and Uncertainty in Climate Data Records from Earth Observations) methodology.

  • Appendix F: AOD Gradient Halo Detection, Debug Diagnostics, Tile Chunking Fix, Terrain Preprocessing, and AOD-Slope Criterion — a two-stage risk mask (Seed OR (Dilated Seed AND Guard)) replacing the single-threshold AOD-gradient criterion for quality-flag Bit 3, the debug=True terrain diagnostic output variables, the HDF5 chunksize clamp fix for narrow edge tiles, configurable terrain slope/aspect preprocessing (interpolated-DEM, Horn/Florinsky method, optional Gaussian pre-smoothing), a DEM no-data nearest-valid-fill fix, a terrain tile-boundary halo fix, and a new AOD-slope criterion for Bit 3.
  • An enhancement note on quality-flag Bit 3 (High AOD gradient): MERRA AOD high-value thresholding alone leaves gradient “halos” around high-AOD pixels that are under-flagged at their core; the implementation now supports a native-MERRA-grid gradient source, a robust (median/MAD-based) threshold option, and an optional morphological dilation+guard step. The robust threshold is now computed once per scene from the full cached native-grid gradient (instead of per tile), and a separately-calibrated aodmax_grad_native constant is used for the fixed method when aod_grad_source=merra_native. See Appendix F.1 and the note under Effect/Bit 3 in the Level 2A Output Variables section.
  • A correction to quality-flag Bit 1 (High AOD): the criterion now compares the MERRA-2 interpolated AOD@550 actually used in the retrieval (AOD_550_used, internally TOTEXTTAU) against aodmax, instead of the maximum TOA reflectance across bands, which was only a loose proxy for aerosol loading. See the note under Effect/Bit 1 in the Level 2A Output Variables section.
  • A new debug configuration option ([Options] debug, default False) that adds three diagnostic output variables — terrain_slope_degrees, terrain_elevation_m, terrain_delta_elevation_m — to help diagnose terrain-uncertainty behavior. See Appendix F.2.
  • A bug-fix note on save_nc_batch (in_out.py): HDF5 chunk sizes are now clamped to the tile’s actual height/width, fixing a ValueError: chunksize cannot exceed dimension size crash on narrow edge tiles smaller than the configured chunk size. See Appendix F.3.
  • Terrain preprocessing controls: clarification that terrain slope/aspect used in slope_err are computed on the DEM interpolated on the sensor grid (not the native DEM grid), plus new configurable [Options] controls — terrain_slope_method (Horn or Florinsky) and terrain_gaussian_sigma (optional Gaussian pre-smoothing of the interpolated DEM before slope/aspect) — and removal of a hardcoded slope resolution constant in favor of the configured sensor_spatial_resolution_m. See Appendix F.4.
  • DEM no-data fill fix: invalid/no-data DEM pixels (ocean, lakes, DEM gaps) are now filled with the value of their nearest valid neighbor instead of a single global constant before computing slope/aspect/Delev, removing spurious stripe artefacts in uncertainty_from_pressure and ring/arc artefacts in uncertainty_from_RTM_terrain that a global-constant fill had created at the boundary of every invalid region. See Appendix F.5.
  • Terrain tile-boundary fix: the optional Gaussian pre-smoothing of the interpolated DEM (terrain_gaussian_sigma) is now applied with a per-tile halo sized to the configured sigma, instead of a fixed 1-pixel halo, removing a spurious periodic discontinuity previously introduced at every processing tile boundary. A persistent terrain-uncertainty hotspot investigated in the same review was confirmed to be a real mountain ridge (elevation up to ~2620 m, a smooth physically-coherent slope profile), not an artefact. See Appendix F.6.
  • New AOD-slope criterion for quality-flag Bit 3: the existing gradient-based Bit 3 criterion uses a coarse native-MERRA-grid gradient that only loosely traces the true AOD transition on the fine sensor grid (measured recall of the true aodmax boundary: ≈11%). A new criterion flags wherever the fine sensor-grid AOD slope exceeds a scene-adaptive threshold, so the flagged region naturally covers the full, variable width of the transition zone (median flagged-band width 1 pixel, 90th-percentile 11 pixels, up to 566 pixels for the broadest gradual transition), increasing total Bit 3 coverage from 125,352 to 589,099 pixels on the validation scene. See Appendix F.7.

Level 1C Input Data

The data used as inputs are Proba-V-VGT1-VGT2 Top-Of-Atmosphere reflectances in 4 spectral bands (Table 1) and associated geometrical information and basic classification flags. Let us note that the viewing geometry could be different between VIS-NIR and SWIR bands. The RSRF’s of each band is slightly different between the 3 cameras labelled LEFT, CENTER and RIGHT that cover Proba-V field of view but one RSRF’s for VGTs. The philosophy of the AC algorithm is to process the data band per band without the use of any spectral, spatial or temporal features. Therefore, the format of the input data is not important and the processing of individual pixels for selected bands or an entire Level 1 tile is similar.

Digital Elevation Model (DEM)

The GTOPO30 (Global 30 Arc-Second Elevation) dataset is used for assigning a surface elevation to each pixel. It is downloaded as a set of binary files (https://lta.cr.usgs.gov/GTOPO30) and then re-assembled as a unique NetCDF file.

  • Frequency: Static

  • Spatial Grid: 2D, full extent of latitude from 90 degrees south to 90 degrees north, and the full extent of longitude from 180 degrees west to 180 degrees east. The horizontal grid spacing is 30-arc seconds.

  • Dimensions of 21,600 rows and 43,200 columns. The horizontal coordinate system is decimal degrees of latitude and longitude referenced to WGS84.

  • The vertical units represent elevation above mean sea level (elev) and are expressed in meters. The values range from -407 to 8,752.

Specifications:

  • Spatial resolution: 30 arc-seconds (~1 km)
  • Coverage: Global (-90° to +90° lat, -180° to +180° lon)
  • Dimensions: 21,600 rows × 43,200 columns
  • Vertical range: -407 to +8,752 m above mean sea level
  • Derived field in processing: Delev - altitude uncertainty estimated from local maximum slope and geolocation error.

Usage:

z = elev(lat, lon)           # Surface elevation
Δz = 0.5 * pixel_size * tan(max_local_slope)  # Elevation uncertainty

Implementation Note: the implementation uses linear interpolation (via scipy.interpolate.RegularGridInterpolator) to sample elev at the satellite grid, then derives Delev from elev only using a local maximum slope and a geolocation uncertainty of 0.5 pixel multiplied by the sensor spatial resolution. This replaces reliance on an external DEM-provided Delev layer. The global DEM is also cropped to the product’s lat/lon extent before loading (~1.9 GB → ~0.2 GB), see Appendix D.1.

SMAC Algorithm Overview

Pixels are considered for atmospheric correction if they are not marked as “cloud-covered” or “contaminated” in the cloud mask of input data. TOC directional reflectance estimates are obtained by applying SMAC (Rahman and Dedieu, 1994), a Simplified Method for the Atmospheric Correction of satellite measurements in the solar spectrum to the TOA reflectances.

The choice of the SMAC algorithm is supported by the following arguments:

  • It is operational and largely used in the land community and already implemented in the Copernicus Global Land Service and Proba-V processing lines.

  • It is a robust and generic algorithm; thus, it minimizes the dependence on the sensor which is a good thing when one wants to build a multi sensor long time series with limited biases.

  • The correction algorithm is based on the 6S radiative transfer formulation of the satellite signal (Vermote et al., 1997) and is analytical It is thus adapted to an error propagation analysis.

In our implementation, all the pertinent radiative quantities are parameterized as a function of auxiliary gridded data as described in the section 2:

  • Gas content (mainly ozone and water vapor for the spectral channels considered in this project)

  • Aerosol content and aerosol type

  • Molecular scattering mainly driven by the sea-level surface pressure and the surface elevation.

This approach is justified by increasing accuracy of the Reanalysis datasets like MERRA-2 or CAMS, which are not sensors biased, without spatial, temporal or spectral gaps. It is particularly well adapted to the reprocessing of Fundamental Climate Data Records.

Equations

All the parametrization is contained in the SMAC coefficients.

\[T = T_{g}\left( \theta_{s},\theta_{v} \right)T_{sca}\left( \theta_{s} \right)T_{sca}\left( \theta_{v} \right) \tag{2}\]

\[R = R_{toa} - R_{atm}\left( \theta_{s},\theta_{v},\Delta\Phi \right)T_{g}\left( \theta_{s},\theta_{v} \right) \tag{3}\]

\[R_{toc}\left( \theta_{s},\theta_{v},\Delta\Phi \right) = \frac{R}{T + s_{atm}R} \tag{4}\]

Where:

- T: Total atmospheric transmission

- Tg: Gaseous transmission (O₃, H₂O, CO₂, CH₄)

- Tsca: Scattering transmission (Rayleigh + aerosol)

- R: Intermediate corrected reflectance

- Ratm: Atmospheric path radiance

- satm: Atmospheric spherical albedo

Sensitivities

It is possible to derive the atmospherically corrected TOC directional reflectance uncertainty \(\Delta R_{toc}\) from the uncertainties on the TOA reflectance, on the atmospheric reflectances and transmittances, which are themselves derived from the uncertainties on the gaseous content, surface pressure aerosol load and model. The sensitivities (Jacobians) to the inputs are obtained in the following manner

Sensitivity to TOA reflectance

The radiometric uncertainty of the TOA reflectance is\(\ \mathrm{\Delta}R_{toa}\). The resulting uncertainty about the TOC directional reflectance\(\ \mathrm{\Delta}_{R_{toc}}^{R_{toa}}\), is obtained:

\[\mathrm{\Delta}_{R_{toc}}^{R_{toa}} = \left| \frac{\partial R_{toc}}{\partial R_{toa}}.\mathrm{\Delta}R_{toa} \right| \tag{5}\]

where \(\frac{\partial R_{toc}}{\partial R_{toa}}\) is the sensitivity of TOC directional reflectance to any change in TOA reflectance. It is the Jacobian\(\ J_{R_{toc}}^{R_{toa}}\).

Within SMAC, it can be derived analytically:

\[J_{R_{toc}}^{R_{toa}} = \frac{\partial R_{toc}}{\partial R}\frac{\partial R}{\partial R_{toa}} \tag{6}\]

after remarking that\(\ \frac{\partial R}{\partial R_{toa}} = 1,\) and using Eqs. 2 and 3

\[\eta = \frac{1}{({T + s}_{atm}\ R)} \tag{7}\]

\[J_{R_{toc}}^{R_{toa}} = \eta^{2} T \tag{8}\]

Sensitivity to Ozone and Water Vapor

The sensitivity of TOC directional reflectance to the uncertainty in the total column of either ozone or water vapor is also treated analytically within SMAC. In the following, we applied the equation for gas X, with X being O3 or H2O.

The uncertainty in the total column \(U_{X}\) is propagated through the Jacobian\(\ J_{R_{toc}}^{U_{X}}\).

\[\mathrm{\Delta}_{R_{toc}}^{U_{X}} = \left| J_{R_{toc}}^{U_{X}}.\mathrm{\Delta}U_{X} \right| \tag{9}\]

Within SMAC, it can also be derived analytically.

Let us recall the formulation of the transmission of the gas \(T_{X}\):

\[{T_{X} = e}^{{{\ a_{X}(U}_{X}m)}^{n_{X}}} \tag{10}\]

where \(a_{X}\) and \(n_{X}\) are SMAC coefficients and m is the air mass:

\[m = \frac{1}{\cos\theta_{s}} + \frac{1}{\cos\theta_{v}} \tag{11}\]

We decompose Jacobian into two parts:

\[J_{R_{toc}}^{U_{X}} = \frac{\partial R_{toc}}{\partial T_{X}}.\frac{\partial T_{X}}{\partial U_{X}} \tag{12}\]

The sensitivity of \(T_{X}\) to \(U_{X}\) is:

\[\frac{\partial T_{X}}{\partial U_{X}} = {\left( \frac{a_{X}.n_{X}}{U_{X}} \right).{\ (U}_{X}m)}^{n_{X}}.T_{X} \tag{13}\]

The sensitivity of TOC directional reflectance to \(T_{X}\) is obtained through Eqs. and using the decomposition\({\ T}_{g} = T_{X}{\ .T}_{noX}\):

\[\frac{\partial R_{toc}}{\partial T_{X}} = {- \eta}^{2}\frac{TR_{toa}}{T_{X}} \tag{14}\]

Surface Pressure and AOT at 550 nm

For the surface pressure \({\ P}_{s}\) and the AOT at 550 nm\({\ \tau}_{a}^{550}\), we process the pixel two times more with small perturbations \(\delta P_{s}\) and\(\ \delta\tau_{a}^{550}\), and then derive the Jacobians using the finite differences:

\[J_{R_{toc}}^{P_{s}} = \frac{{R_{toc}\left( P_{s} \right) - \rho}_{toc}\left( P_{s} - \delta P_{s} \right)}{\delta P_{s}} \tag{15}\]

\[\mathrm{\Delta}_{R_{toc}}^{P_{s}} = \left| J_{R_{toc}}^{P_{s}}.\mathrm{\Delta}P_{s} \right| \tag{16}\]

\[J_{R_{toc}}^{\tau_{a}^{550}} = \frac{{R_{toc}\left( \tau_{a}^{550} \right) - R}_{toc}\left( \tau_{a}^{550} - \delta\tau_{a}^{550} \right)}{\delta\tau_{a}^{550}} \tag{17}\]

\[\mathrm{\Delta}_{R_{toc}}^{\tau_{a}^{550}} = \left| J_{R_{toc}}^{\tau_{a}^{550}}.\mathrm{\Delta}\tau_{a}^{550} \right| \tag{18}\]

We use\(\ \delta P_{s} = 10\ hPa,\ \delta\tau_{a}^{550} = 0.1\ \tau_{a}^{550}\). These values are chosen to provoke a sufficient response in the AC scheme and still stay in the linear regime

Primary Measurement Equation

The TOC directional reflectance for each spectral band j is obtained through the SMAC atmospheric correction:

\[\begin{aligned} R_{toc}\left( \theta_{s},\theta_{v},\Delta\Phi \right) &= f\left( R_{toa},T_{g},T_{sca},R_{atm},s_{atm};U_{H_{2}O},U_{O_{3}},P_{s},\tau_{a}^{550},aerosol\_ model \right) \\ &\quad + \varepsilon_{0} \end{aligned} \tag{19}\]

Where:

- Rtoc: Top-of-Canopy directional reflectance (output)

- Rtoa: Top-of-Atmosphere reflectance (primary input)

- Tg: Total gaseous transmission

- Tsca: Atmospheric scattering transmission

- Ratm: Atmospheric path reflectance

- satm: Atmospheric spherical albedo

- UH₂O: Total column water vapor (auxiliary input from MERRA-2)

- UO₃: Total column ozone (auxiliary input from MERRA-2)

- Ps: Surface pressure (derived from DEM and MERRA-2 SLP)

- τa⁵⁵⁰: Aerosol optical thickness at 550 nm (auxiliary input from MERRA-2)

- aerosol model: Selected aerosol optical properties

- ε₀: Second-order uncertainty terms

Primary Uncertainty Equation

The total TOC reflectance uncertainty is derived from the measurement equation through uncertainty propagation:

\[u\left( R_{toc}^{j} \right) = \sqrt{\begin{aligned} &\left( \frac{\partial R_{toc}^{j}}{\partial R_{toa}^{j}}u(R_{toa}^{j}) \right)^{2} + \left( \frac{\partial R_{toc}^{j}}{\partial U_{H_2O}}u(U_{H_2O}) \right)^{2} \\ &+ \left( \frac{\partial R_{toc}^{j}}{\partial U_{O_3}}u(U_{O_3}) \right)^{2} + \left( \frac{\partial R_{toc}^{j}}{\partial P_{S}}u(P_{S}) \right)^{2} \\ &+ \left( \frac{\partial R_{toc}^{j}}{\partial \tau_{a}^{550}}u\left( \tau_{a}^{550} \right) \right)^{2} + \left( u\left( R_{toc}^{j} \right)_{ensemble} \right)^{2} \\ &+ \left( u\left( R_{toc}^{j} \right)_{RTM} \right)^{2} \end{aligned}} + u^{2}(0) \tag{20}\]

Where individual contributions include:

  • TOA reflectance uncertainty (from Level 1C, propagated with analytical Jacobians)
  • Gaseous absorption uncertainties (O₃, H₂O, propagated with analytical Jacobians)
  • Surface pressure uncertainty (from DEM + geolocation, propagated with finite difference)
  • Aerosol optical depth uncertainty (from MERRA-2 validation studies, propagated with finite difference)
  • Aerosol model uncertainty (from AMIP ensemble spread)
  • Radiative transfer approximation errors (SMAC vs. exact RT, terrain slope, adjacency…)
  • Second-order effects (u²(0) term)

Surface Pressure Uncertainty

The uncertainty on the surface pressure \(P_{S}\) is due to the uncertainty on the meteorological sea level pressure P0 from MERRA-2 SLP parameter (very low, <1 hPa), and the way the extrapolation to the altitude of the surface z is done. z is obtained through GTOPO30 elev, which has a resolution close to the targeted satellite product. The uncertainty in the mean altitude of the pixel is ∆z.

The surface pressure Ps is derived as:

\[P_{S} = P_{0}\ e^{- \frac{g}{r\lambda}T^{'}} \tag{21}\]

\[T^{'} = \ln(rT) - \ln(rT - r\lambda z) \tag{22}\]

Where T is the temperature above ground (from MERRA-2 T10M parameter), g is Earth’s gravity (9.80665 m.s−2), r is the gas constant for dry air (287.058 J.kg−1.K−1) and λ is the lapse rate of the atmosphere, here taken as −0.006 K.m−1 (Van Besselaar et al., 2011). The accuracy of this model depends on the variability of λ (typically ∆λ = 0.002 K.m−1). An uncertainty of \(\mathrm{\Delta}_{P_{s}}^{\lambda}\)~1hPa could be assigned to this modeling error.

However, the uncertainty in the geolocation could result in a significant uncertainty ∆z, especially in rough terrain. The latter error could be partly a random error and a pixel dependent bias.

For example, if the geolocation error is purely random and isotropic, with a stable standard deviation, one may estimate the altitude error from local terrain slope and geolocation error.

Figure 2: Surface elevation (left) and local standard deviation of altitude (right), both in m, from GTOPO30.

The implementation derives Delev directly from elev and local maximum slope, using a geolocation error of 0.5 pixel and the sensor spatial resolution:

\[u(z) = \Delta z = \Delta r \cdot \tan\left(\theta_{\max}\right)\]

where \(\Delta r = 0.5 \times \text{pixel\_size}\) and \(\theta_{\max}\) is the local maximum slope used for terrain uncertainty estimation.

Finally, the uncertainty on Ps is:

\[u(P_{s}) = \sqrt{\left\lbrack \left( \mathrm{\Delta}_{P_{s}}^{\lambda} \right)^{2} + \left( \mathrm{\Delta}_{P_{s}}^{z} \right)^{2} \right\rbrack/2} \tag{23}\]

with

\[\mathrm{\Delta}_{P_{s}}^{z} = \left| J_{P_{s}}^{z}.u(z) \right| \tag{24}\]

and

\[J_{P_{s}}^{z} = - \frac{gP_{S}}{r(T - \lambda z)} \tag{25}\]

Aerosol Optical Depth Uncertainty

Quantifying aerosol optical depth (AOD) uncertainty in global chemical transport models (CTMs) remains critical for climate and air quality applications. Recent studies have highlighted systematic biases and uncertainties in leading reanalysis products such as CAMS and MERRA-2.

Comparative validation studies against AERONET ground-based observations reveal that both CAMS and MERRA-2 exhibit regional biases, with typical AOD uncertainties ranging from 30-50% depending on aerosol type and geographic location. Model-observation discrepancies are particularly pronounced over dust-dominated regions and biomass burning areas, where AOD can be underestimated by factors of 1.5-2.0.

Advances in uncertainty quantification have emerged through ensemble approaches and data assimilation techniques. Studies show that assimilating satellite retrievals can reduce AOD uncertainty by 20-35%, though systematic errors in aerosol vertical distribution persist. Emission inventory uncertainties, particularly for anthropogenic sources in rapidly developing regions, can account for 25-40% of total AOD uncertainty.

The community is moving toward probabilistic frameworks that explicitly quantify measurement uncertainties, model uncertainties, and structural uncertainties arising from parameterization choices in aerosol microphysics, chemistry, and removal processes.

The AOD uncertainties (Root Mean Square Errors) could be assigned on a regional scale using a climate type classification map like the one used in Gueymard and Yang 2020. Years of AOD @550 and Angström exponent 3-hourly datasets from MERRA-2 and CAMS have been evaluated at 793 AERONET stations. The validation was conducted based on continents and Köppen-Geiger classification (see Figure 3).

RMSE Model

Based on the Gueymard and Yang findings, a practical operational uncertainty predictor can be formulated as:

\[\sigma\left( \tau_{a}^{550} \right) = \max\left( \sigma_{base},\sigma_{relative} \times \tau_{a}^{550} \right) \tag{26}\] Where:

- σbase = 0.03 (clean atmosphere baseline)

- σrelative = 0.15 (15% relative uncertainty)

This is justified by the following arguments

- Low AOD regime (τa < 0.2): Dominated by instrument noise ~0.03

- Moderate-high AOD (τa ≥ 0.2): Scales with aerosol loading → 15% uncertainty

Terrain Slope Effects

Mountain areas, rough terrain, have been modelled and estimated through a simple model following cesbio (https://www.cesbio.cnrs.fr/multitemp/2481/)

The slope_eff function computes the terrain-induced relative effect. Here are the mathematical equations:

Input Variables:

  • \(\theta_{s}\) : Solar Zenith Angle (SZA)

  • \(\phi_{s}\) : Solar Azimuth Angle (SAA)

  • \(s\) : Terrain slope (computed from DEM using xdem python package)

  • \(\phi_{a}\) : Terrain aspect (computed from DEM using xdem python package)

Derived Quantities:

  • Cosine of solar zenith angle:

\[\mu_{s} = cos\left( \theta_{s} \right) \tag{27}\] - Cosine of terrain slope:

\[\mu = cos(s) \tag{28}\] - Effective cosine of incidence angle on tilted surface:

\[\begin{aligned} \mu_{i} &= \mu_{s} \cdot \mu \\ &\quad + \sqrt{1 - \mu_{s}^{2}} \cdot \sqrt{1 - \mu^{2}} \cdot cos\left( \phi_{s} - \phi_{a} \right) \end{aligned} \tag{29}\]

  • Sky view factor:

\[f_{sky} = \frac{1 + \mu}{2} \tag{30}\] - Ground view factor:

\[f_{ground} = \frac{1 - \mu}{2} \tag{31}\] - Slope Error

\[slope_{eff} = \frac{T}{ T_{dir} \cdot \max\left(0,\frac{\mu_{i}}{\mu_{s}}\right) + T_{dif} \cdot f_{sky} + T \cdot f_{ground} \cdot \rho_{env} } \tag{32}\]

with the direct term explicitly set to zero for terrain facets facing away from the sun (\(\mu_i \le 0\)), so it never contributes a negative correction.

  • \({u\left( R_{toc}^{j} \right)}_{RTM - slope} = \left|R_{toc}^{j}*(slope_{eff} - 1)\right|\)

Where:

\(T\) : Total atmospheric transmittance (an output of the SMAC function)

\(T_{dir}\) : Direct atmospheric transmittance (an output of the SMAC function)

\(T_{dif}\) : Diffuse atmospheric transmittance (an output of the SMAC function)

\(\rho_{env}:\) Average environment reflectance, (supposed here equal to pixel ground reflectance)

Limitations and Future Improvements

Known Limitations

Current Implementation:

  • Surface BRDF: Simplified kernel model
    • Impact: Up to 10% error in directional reflectance over non-forest surfaces
  • Terrain effect: Simplified model
  • Water Vapor Uncertainty: Conservative 20% estimate
    • Impact: May overestimate total uncertainty in dry conditions
    • Future: Refined characterization from MERRA-2 validation studies
  • Adjacency Effects: Not corrected
    • Impact: Significant for high-contrast scenes (coast, cloud edge)
    • Option: Implement semi-empirical adjacency correction

Status update: Adjacency effects remain a confirmed known limitation (not corrected in the operational chain). Köppen-Geiger regional AOD uncertainty multipliers (Section 5.3.5) are now implemented and applied to the AOD uncertainty used in uncertainty propagation.

Conclusions

This ATBD demonstrates a comprehensive implementation of the FIDUCEO uncertainty propagation methodology for atmospheric correction of satellite vegetation sensors. Key achievements:

  • Rigorous Uncertainty Quantification: All major uncertainty sources are identified, characterized, and propagated through the measurement equation

  • Operational AOD Uncertainty Model: Based on extensive MERRA-2 validation (Gueymard & Yang 2020), providing realistic AOD uncertainties: σ(τa) = max(0.03, 0.15×τa) with regional adjustments

  • Ensemble Approach for Model Uncertainty: AMIP 10-member ensemble quantifies aerosol optical property uncertainties

  • Efficient GPU Implementation: SMAC-CL enables fast processing while computing full Jacobian matrices

  • Validation Framework: Following Aerosol_cci and FIDUCEO best practices (Popp & Mittaz 2022)

This approach provides climate-quality TOC reflectances with fully traceable, metrologically sound uncertainties suitable for:

  • Long-term surface monitoring

  • Multi-sensor harmonization

  • Climate model validation

BRDF effect estimation

The baseline AC are run under the assumption of a lambertian surface. If the shape of the surface BRDF is known, we can optionally include this knowledge into the AC scheme with the following modelling

Surface BRDF model (RossThick-LiSparse-R)

In the following, we restrict the description to a model with three parameters of the form:

\[\rho_{g}(\theta_{s}, \theta_{v}, \phi) = k_{0} + k_{1}F_{1}(\theta_{s}, \theta_{v}, \phi) + k_{2}F_{2}(\theta_{s}, \theta_{v}, \phi) \tag{33}\]

While k0 quantifies an isotropic contribution to the reflectance factor F0=1, functions F1 and F2 represent the angular distribution related to geometric and volumetric surface scattering processes, respectively. Lucht et al. (2000) suggest the RossThick-LiSparse-Reciprocal model (simply referred to as the Ross-Li BRDF model) that results from the combination of two models for the definition of the volumetric (RossThick) and geometric (LiSparse-R) kernels. The geometric kernel is expressed as follows:

\[F_{1}\left( \theta_{s}{,\theta}_{v},\phi \right) = O\left( \theta_{s}{,\theta}_{v},\phi \right) - sec\left( {\theta\ '}_{s} \right) - sec\left( {\theta\ }_{v}^{'} \right) + \frac{1}{2}\left( 1 + cos(\xi\ ') \right)\sec\left( {\theta\ '}_{s} \right)\sec\left( {\theta\ }_{v}^{'} \right) \tag{34}\]

Where,

\[O = \frac{1}{\pi}\left( t - sin(t)\cos(t) \right)\left( \sec\left( \theta_{s}^{\ '} \right) - sec\left( \theta_{v}^{\ '} \right) \right) \tag{35}\]

\[\cos(t) = 2\frac{\sqrt{D^{2} + \left( \tan\left( \theta_{s}^{\ '} \right)\tan\left( \theta_{v}^{\ '} \right)\sin(\phi) \right)^{2}}}{\sec\left( \theta_{s}^{\ '} \right) + sec\left( \theta_{v}^{\ '} \right)} \tag{36}\]

\[D = \sqrt{\tan^{2}\left( \theta_{s}^{\ '} \right) + \tan^{2}\left( \theta_{v}^{\ '} \right) - 2tan\left( \theta_{s}^{\ '} \right)\tan\left( \theta_{v}^{\ '} \right)\cos(\phi)} \tag{37}\]

\[\cos\left( \xi^{\ '} \right) = cos\left( \theta_{s}^{\ '} \right)\cos\left( \theta_{v}^{\ '} \right) + sin\left( \theta_{s}^{\ '} \right)\sin\left( \theta_{v}^{\ '} \right)\cos(\phi) \tag{38}\]

\[\theta_{s}^{\ '} = \tan^{- 1}\left( \frac{b_{RL}}{r}\tan\theta_{s} \right),\quad \theta_{v}^{\ '} = \tan^{- 1}\left( \frac{b_{RL}}{r}\tan\theta_{v} \right) \tag{39}\]

And the volumetric kernel is defined as:

\[F_{2}\left( \theta_{s}{,\theta}_{v},\phi \right) = \frac{\left( \frac{\pi}{2} - \xi \right)\cos(\xi) + sin(\xi)}{\cos\left( \theta_{s} \right) + cos\left( \theta_{v} \right)} - \frac{\pi}{4} \tag{40}\]

Where

\[\cos\left( \xi^{\ } \right) = cos\left( \theta_{s}^{\ } \right)\cos\left( \theta_{v}^{\ } \right) + sin\left( \theta_{s}^{\ } \right)\sin\left( \theta_{v}^{\ } \right)\cos(\phi) \tag{41}\]

The choice is used here: h/bRL=2 and bRL/r=1.

We can summarize the surface BRDF modelling as:

\[\rho_{g} = k_{0}\left\lbrack 1 + k_{1}^{p}F_{1} + k_{2}^{p}F_{2} \right\rbrack \tag{42}\]

with \(k_{1}^{p} = \frac{k_{1}}{k_{0}} ;\ k_{2}^{p} = \frac{k_{2}}{k_{0}}\)

kp’s are normalized coefficients that describe the departure from the lambertian reflector (in that case\({\ k}_{1}^{p} = k_{2}^{p} = 0\)), this is the baseline processing.

Fast radiative transfer model with BRDF

In the case of a horizontally homogeneous target, the surface BRDF shape can be included through the 6S formulation (Vermote et al., 1997), and the atmospheric correction equations [2-4] are the same except that we have a modified total atmospheric transmission \(T_{atm;g}\)(Ramon and Santer, 2001):

\[\begin{aligned} T_{atm;g} &= \left( 1 - D\left( \mu_{v} \right) \right)\left( 1 - D\left( \mu_{s} \right) \right) + \frac{\overline{\rho_{g}}}{\rho_{g}}\left( 1 - D\left( \mu_{v} \right) \right)D\left( \mu_{s} \right) \\ &\quad + \frac{\overline{\rho_{g}}'}{\rho_{g}}D\left( \mu_{v} \right)\left( 1 - D\left( \mu_{s} \right) \right) + \frac{\overline{\overline{\rho_{g}}}}{\rho_{g}}D\left( \mu_{v} \right)D\left( \mu_{s} \right) \end{aligned} \tag{43}\]

Where D is the ratio of diffuse over total transmission and \(\ \overline{\rho_{g}}\ \ and\ \overline{\overline{\rho_{g}}}\) are single and double directional weighted averages of the surface BRDF. The weight is the atmospheric downward radiance\(\ L_{atm}^{\downarrow}\) at the ground level:

\[\begin{aligned} \overline{\rho_{g}} &= \overline{\rho_{g}}(\theta_{s}{,\theta}_{v},\phi) \\ &= \frac{\int_{0}^{2\pi}{\int_{0}^{\frac{\pi}{2}}{\cos\theta^{'}L_{atm}^{\downarrow}\left( \theta_{s}{,\ \theta}^{'},\varphi^{'} \right)\rho_{g}\left( \theta^{'},\theta_{v},\varphi^{'} - \phi \right)\sin{\theta^{'}d\theta^{'}d\varphi^{'}}}}}{\int_{0}^{2\pi}{\int_{0}^{\frac{\pi}{2}}{\cos{\theta^{'}L_{atm}^{\downarrow}\left( \theta_{s}{,\ \theta}^{'},\varphi^{'} \right)}}}\sin{\theta^{'}d\theta^{'}d\varphi^{'}}} \end{aligned} \tag{44}\]

where \({\overline{\rho_{g}}}^{'}\left( \theta_{s}{,\theta}_{v},\phi \right) = \overline{\rho_{g}}(\theta_{v}{,\theta}_{s},\phi)\) and

\[\overline{\overline{\rho_{g}}} = \overline{\overline{\rho_{g}}}\left( \theta_{s}{,\theta}_{v},\phi \right) = \overline{{\overline{\rho_{g}}}^{'}\left( \theta_{s}{,\theta}_{v},\phi \right)} \tag{45}\]

The linearization of the ground reflectance allows the ground to derive depending on terms as a function of just kernel directional averages \(\overline{F_{1}},\overline{F_{2}},\overline{\overline{F_{1}}},\overline{\overline{F_{2}}}\):

\[\begin{aligned} \overline{\rho_{g}} &= k_{0}\left\lbrack 1 + k_{1}^{p}\overline{F_{1}} + k_{2}^{p}\overline{F_{2}} \right\rbrack \\ \overline{\overline{\rho_{g}}} &= k_{0}\left\lbrack 1 + k_{1}^{p}\overline{\overline{F_{1}}} + k_{2}^{p}\overline{\overline{F_{2}}} \right\rbrack \end{aligned} \tag{46}\]

\[\frac{\overline{\rho_{g}}}{\rho_{g}} = \frac{1 + k_{1}^{p}{\overline{F}}_{1} + k_{2}^{p}{\overline{F}}_{2}}{1 + k_{1}^{p}F_{1} + k_{2}^{p}F_{2}} \tag{47}\]

\[\frac{\overline{\rho_{g}}'}{\rho_{g}} = \frac{1 + k_{1}^{p}{\overline{F}}_{1}' + k_{2}^{p}{\overline{F}}_{2}'}{1 + k_{1}^{p}F_{1} + k_{2}^{p}F_{2}} \tag{48}\]

\[\frac{\overline{\overline{\rho_{g}}}}{\rho_{g}} = \frac{1 + k_{1}^{p}\overline{\overline{F_{1}}} + k_{2}^{p}{\overline{\overline{F}}}_{2}}{1 + k_{1}^{p}F_{1} + k_{2}^{p}F_{2}} \tag{49}\]

The estimation of BRDF effects requires the knowledge of \(k_{1}^{p}\) and\({\ k}_{2}^{p}\), the kernels and their directional averages are analytically computed

SMAC general philosophy is to approximate atmospheric functions with simple and fast regression. We thus analysed the variations of the kernel directional averages as a function of wavelength and aerosol optical thickness. The computation has been made with exact radiative transfer computations for the downward atmospheric radiances and integration using Equations (38). Figure 12 and Figure 13 show some examples of \(\overline{{\ F}_{1}},\overline{F_{2}},\overline{\overline{F_{1}}},\overline{\overline{F_{2}}}\) in the NIR for one aerosol model and 2 AOTs as well as the case where the sky radiance is supposed isotropic. In the latter case the kernel directional single (resp. double) integral is simply the hemispherical-directional (resp. bi hemispherical) albedo. Directionality is still pronounced for single integrals, and smooth for double ones. It is mainly dependent on wavelength and not AOT and the directional behaviour is like the kernels themselves. So the idea here is to make a computation of \(\overline{{\ F}_{1}},\overline{F_{2}},\overline{\overline{F_{1}}},\overline{\overline{F_{2}}}\) as linear combinations of the kernels F1 and F2, the coefficients of the combinations being only functions of the spectral channel.

\[\begin{aligned} {\overline{F}}_{1} &= d_{1}^{0} + d_{1}^{1}\mathbf{F}_{\mathbf{1}} + d_{1}^{2}\mathbf{F}_{\mathbf{2}} \\ {\overline{F}}_{2} &= d_{2}^{0} + d_{2}^{1}\mathbf{F}_{\mathbf{1}} + d_{2}^{2}\mathbf{F}_{\mathbf{2}} \\ \overline{\overline{F_{1}}} &= b_{1}^{0} + b_{1}^{1}\mathbf{F}_{\mathbf{1}} + b_{1}^{2}\mathbf{F}_{\mathbf{2}} \\ {\overline{\overline{F}}}_{2} &= b_{2}^{0} + b_{2}^{1}\mathbf{F}_{\mathbf{1}} + b_{2}^{2}\mathbf{F}_{\mathbf{2}} \\ {\overline{F}}_{1}^{'} &= d_{1}^{0} + d_{1}^{1}\mathbf{F}_{\mathbf{1}}^{'} + d_{1}^{2}\mathbf{F}_{\mathbf{2}}^{'} \\ {\overline{F}}_{2}^{'} &= d_{2}^{0} + d_{2}^{1}\mathbf{F}_{\mathbf{1}}^{'} + d_{2}^{2}\mathbf{F}_{\mathbf{2}}^{'} \end{aligned} \tag{50}\]

Where, by convention:

\[\begin{aligned} \mathbf{F}_{1} &= F_{1}\left( \theta_{s}{,\theta}_{v},\phi \right) \\ \mathbf{F}_{2} &= F_{2}\left( \theta_{s}{,\theta}_{v},\phi \right) \\ \mathbf{F}_{1}' &= F_{1}\left( \theta_{v}{,\theta}_{s},\phi \right) \\ \mathbf{F}_{2}' &= F_{2}\left( \theta_{v}{,\theta}_{s},\phi \right) \end{aligned} \tag{51}\]

The 12 regressions coefficients \(d_{i}^{k}\) and \(b_{i}^{k}\) are stored in the SMAC coefficient’s structure. F1 and F2 are analytical functions, computed once per pixel. Therefore, the corrective BRDF terms are calculated extremely fast.

The accuracy of such modelling has been checked against exact computations and compared to the accuracy of the lambertian assumption. A typical example is given in Figure 14 and Figure 15 for a scene that consists of a vegetation target in the NIR with a high aerosol load, where the BRDF effects are the highest. On can see that the lambertian assumption leads to directional biases that could reach 10% for oblique views and SZA=60°. Our simple model significantly reduces the intensity and directional dependence of those biases.

44 688 MERRA-2 grid points have been selected over land. Four dates have been chosen to represent the seasonal variability: 2011/01/01, 2011/04/01, 2011/07/01, and 2011/10/01. Data is taken at 13:00.

    • Bit 4 : AC algorithm failure

    • Bit 5 : Missing auxiliary data

    • Bit 6 : Out of LUT range

For implementation consistency, a single AOD threshold parameter aodmax is used in configuration and propagated to the SMAC-CL internal high-AOD threshold (taot) at runtime. Therefore, both the internal SMAC-CL high-AOD process flag and the project quality criterion use the same numerical limit.

Implementation Note (v1.1): Bit 1 now compares the MERRA-2 interpolated AOD@550 actually used in the retrieval (TOTEXTTAU, output as AOD_550_used) against aodmax, rather than the maximum TOA reflectance across bands. The reflectance-based criterion was only an indirect proxy for aerosol loading (reflectance is also driven by surface type, sun-view geometry, etc.), whereas the AOD-based criterion directly reflects the quantity the threshold name (“High AOD”) describes and is consistent with the input used elsewhere in the aerosol-correction chain.

The gradient-based flag uses:

\[aod\_grad = \sqrt{\left(\frac{\partial \tau_{a}^{550}}{\partial y}\right)^2 + \left(\frac{\partial \tau_{a}^{550}}{\partial x}\right)^2}\]

and raises Bit 3 when:

\[aod\_grad > aodmax\_grad\]

Implementation Note (v1.1): MERRA AOD high-value thresholding alone leaves gradient “halos” of too-high Rtoc around high-AOD pixels, since the strongest gradient is often only at the halo edge, not the core. The implementation therefore supports an enhanced, two-stage risk mask for Bit 3: Bit3 = Seed OR (DilatedSeed AND Guard), where Seed is the aod_grad > aodmax_grad criterion above (optionally on a robust median/MAD-based threshold instead of a fixed one), DilatedSeed is a morphological dilation (+ optional closing) of Seed to catch adjacent halo-ring pixels, and Guard requires a moderate relative gradient or elevated relative AOD to prevent overgrowth. See Appendix F.1 for the configuration knobs and validation results.

Unimplemented Features and Future Work

Adjacency Effects

Status: Not corrected. Confirmed known limitation.

Impact: significant for high-contrast scenes (coast, cloud edge, urban/rural boundaries). The SMAC algorithm assumes a horizontally homogeneous scene within the pixel footprint; adjacent pixels with different reflectances contribute to the measured TOA radiance through atmospheric scattering, which is not accounted for.

Future work: implement a semi-empirical adjacency correction (e.g. based on scene contrast and atmospheric conditions).

Köppen-Geiger Regionalization

Status: Implemented.

The AOD uncertainty model (Section 5.3.5, equation definition) uses a global base relation scaled by regional multipliers derived from Köppen-Geiger climate classes. Multipliers are applied to Dtaup before uncertainty propagation in compute_urtoc():

Tile Processing Architecture

The processing pipeline uses a tile-based architecture for memory-bounded, parallel execution:

process_batched() -- Main entry point
 1. Generate tile grid from input extent
    - Tile size: chunks_size x chunks_size
    - Overlap: +/-1 pixel halo
    - Total tiles: ceil(y/chunks) x ceil(x/chunks)
 2. Precompute ancillary cache (if enabled)
    - Interpolate MERRA-2 to full grid
    - Save to local Zarr cache
 3. Create worker pool (nworkers)
    - Fork workers before populating _SHARED
    - Copy-on-write inheritance of large objects
 4. Process tiles in parallel
    - _worker_compute_tile() per tile
    - SMAC-CL run (VNIR + SWIR)
    - Uncertainty propagation
    - Write tile to NetCDF
 5. Move local output to final destination

_worker_compute_tile() performs the end-to-end tile computation:

  1. Read tile data from the input L1C file and the ancillary cache
  2. Interpolate the DEM and compute terrain (slope/aspect)
  3. Select the aerosol model (closest in 5D species space)
  4. Run SMAC-CL (VNIR bands, then SWIR band)
  5. Compute the BRDF effect (Lambertian vs BRDF run, if enabled)
  6. Compute the terrain effect (CESBIO model)
  7. Compute the RTM fit uncertainty (ONNX model)
  8. Propagate uncertainties (compute_urtoc())
  9. Build quality flags (build_flag())
  10. Write the tile to the output NetCDF

Appendix F: Quality Flags, Terrain Preprocessing, and Diagnostics — Implementation Details

This appendix describes implementation-level behavior controlled by configuration parameters: the Bit 3 quality-flag construction, terrain derivative preprocessing, debug diagnostic outputs, and HDF5 tile chunking. These are engineering details of the operational chain and do not change the underlying AC physics described in Sections 1-10.

F.1 AOD Gradient Halo Detection (Quality-Flag Bit 3)

Bit 3 is raised by a two-stage risk mask:

\[Bit3 = Seed \lor (DilatedSeed \land Guard)\]

  • Seed — \(aod\_grad > aodmax\_grad\), with two threshold variants selected by aod_grad_threshold_method:
    • fixed: a constant threshold. When aod_grad_source=merra_native (gradient computed on the native ~0.5-0.625° MERRA-2 grid), the threshold used is aodmax_grad_native (default 0.15); when aod_grad_source=legacy (gradient of the sensor-grid-interpolated AOD), aodmax_grad (default 1.4e-4) is used.
    • robust (default): \(\max(floor,\ median + k \cdot 1.4826 \cdot MAD,\ quantile(q))\), computed once per scene from the full cached native-grid gradient.
  • Dilated Seed — a morphological dilation (with optional closing) of the Seed mask.
  • Guard — requires either a moderate gradient (relative to aodmax_grad) or an elevated interpolated AOD (relative to a threshold) before a dilated pixel is kept.

build_flag() in fdr4vgt/funcs.py computes the mask; interp_merra_native_aod_grad() in fdr4vgt/process.py computes the native-grid gradient.

F.4 Terrain Derivative Preprocessing Controls

Terrain slope/aspect (used by the terrain-effect uncertainty term) are computed on the DEM interpolated onto the sensor grid (lat, lon of the tile), not the native DEM grid:

  • terrain_slope_method: Horn (3x3, default) or Florinsky, via xdem.terrain.
  • terrain_gaussian_sigma (default 0.0, disabled): optional Gaussian pre-smoothing of the interpolated DEM before slope/aspect, to reduce high-frequency artefacts over low-relief/quantized-elevation areas.
  • sensor_spatial_resolution_m sets the spatial resolution used in the slope computation.

F.5 DEM No-Data Handling for Terrain Derivatives

Invalid/no-data DEM pixels (ocean, lakes, DEM gaps) are filled with the value of their nearest valid neighbor (_nearest_valid_fill(), fdr4vgt/funcs.py, via scipy.ndimage.distance_transform_edt) before computing slope/aspect and Delev, keeping the filled region locally continuous with the surrounding terrain. The final slope/aspect output preserves the original DEM no-data mask (slope[~valid] = np.nan).

F.6 Terrain Derivative Halo Sizing

When terrain_gaussian_sigma > 0, the per-tile lat/lon window used for the terrain slope/aspect computation is widened to \lceil 4 \times \text{sigma} \rceil + 1 pixels of halo (instead of the 1-pixel halo used elsewhere), sliced directly from the full-scene lat/lon grid in _worker_compute_tile() (fdr4vgt/process.py), then cropped back to the tile’s own extent. This ensures the Gaussian kernel has sufficient spatial context at every tile boundary.

F.7 AOD-Slope Criterion for Quality-Flag Bit 3

In addition to the native-grid gradient criterion (Appendix F.1), Bit 3 also includes an AOD-slope criterion based on aod_grad_fine — the gradient magnitude of the AOD already interpolated onto the sensor grid, computed once per scene in precompute_ancillary() (fdr4vgt/process.py) via np.gradient on the full cached AOD_550_used grid. A pixel is flagged when aod_grad_fine exceeds a global, scene-adaptive threshold: \(\max(\text{median} + k \cdot 1.4826 \cdot \text{MAD},\ \text{quantile}(q))\), computed once from the whole-scene grid. This is OR-ed into Bit 3 in build_flag() (fdr4vgt/funcs.py), in addition to (not replacing) the Appendix F.1 criterion.
